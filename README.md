# AFINIT BLOG

> 基于springboot开发的个人博客网站

# 一、所用技术

- 核心应用框架（Core Framework)：SpringBoot，SpringMvc
- 持久层（ORM框架）：MyBatis-Plus
- 数据库驱动：MySQL Connector
- 开发辅助与工具类库：Lombok
- 测试框架：Spring Boot Test
- 项目构建工具：Maven
- 版本控制工具：Git，GitHub
- 数据库： MySQL

# 二、项目结构

## 1、三层架构

该项目严格按照三层架构编写，包括表现层，业务逻辑层，数据访问层

- 表现层：采用`RESTful`风格，负责接收前端网页的各种http请求并进行合法性校验，将合法的请求转交给业务逻辑层。
- 业务逻辑层：处理表现层发来的请求，进行核心业务处理，在设计到数据库相关操作时调用数据访问层封装好的各类方法。
- 数据表现层：负责与数据库进行链接通信，用于对数据库进行数据处理（增、删、改、查），提供封装好的方法供业务逻辑层调用。

## 2、SpringBoot

- 规范开发：以企业常用的`SpringBoot`框架作为基础脚手架，从底层即做好了规范化开发。
- 简化开发：`SpringBoot`提供了自动配置机制，让开发时不必因为各个依赖的版本不兼容而焦头烂额。

## 3、Maven

- 强大的依赖管理：通过`pom.xml`集中声明依赖，Maven会自动从中央仓库下载所需的jar包。它还能自动处理依赖的传递性，大幅减少了手动寻找和导入jar包的工作量，并有效缓解了版本冲突问题。
- **标准化的目录结构**：SpringBoot提供了一套行业通用的标准目录骨架（如`src/main/java`用于源码，`src/test/java`用于测试）
- 统一的构建生命周期：它将项目的构建过程抽象为清理（`clean`）、编译（`compile`）、测试（`test`）、打包（`package`）、安装（install）和部署（`deploy`）等标准阶段。执行一条简单命令即可自动化完成整个流程。

## 4、Git

- 分布式架构：每个开发者的本地都会完整克隆整个代码仓库的历史记录。这意味着绝大多数操作都可以在离线状态下极速完成，且彻底避免了单点故障导致的数据丢失。
- 轻量的分支管理：Git可进行分支创建和切换，这可以为每个模块的开发单独开一个分支，互不干扰，最后再合并到主干

- 强大的版本控制：在每一次提交代码都会有一条新纪录，随时可以查看和回溯代码
